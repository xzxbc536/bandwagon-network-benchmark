## 一、先把"网络性能"这件事说清楚：搬瓦工到底有几条线路

搬瓦工之所以在国内用户圈子里能火这么多年，核心不是配置便宜（其实它的高端套餐并不便宜），而是它把"中国方向网络优化"这件事做成了产品矩阵。你买的不是一台 VPS，而是一条**面向三网优化的回程线路**。

目前搬瓦工在售的线路大致分成这几档（从低到高）：

1. **普通 KVM 直连线路**：走的是国际公共路由，没有专门的中国方向优化，价格最便宜，但晚高峰容易被挤。机房可选 DC2 QNET、DC4 MCOM、DC8 ZNET 等老牌机房。
2. **CN2 GT 线路**：电信 CN2 的入门款，去程走 CN2，回程不一定，延迟比普通直连稍好，是 KVM 套餐下挂载的 DC3 CN2 机房用的就是这条。
3. **CN2 GIA-E 线路（主力推荐）**：电信 AS4809 + 联通 AS10099 + 移动 CMIN2 AS58807 三网优化，DC6 CN2 GIA-E 和 DC9 CN2 GIA 是两个核心机房，是搬瓦工的招牌。
4. **香港 CN2 GIA / 大阪 CN2 GIA**：物理位置就近，延迟更低，但带宽小、价格贵，是"对延迟敏感且预算充足"用户的选择。
5. **SLA 套餐**：在 CN2 GIA-E 基础上加了 99.99% 服务等级协议（Service Level Agreement），目前只有 USCA_5 机房提供，适合对稳定性有合同级要求的商用场景。

理解了这个分层，你就能明白：所谓"搬瓦工网络性能"，**本质上是在说 CN2 GIA-E 这条三网优化线路到底有多能打**。下文所有的实测数据，重点也是围绕这条线路展开。

## 二、CN2 GIA-E 实测：延迟、带宽、丢包的真实数字

### 1. 延迟表现：中美之间几乎已经是物理极限

DC6 CN2 GIA-E 机房是搬瓦工 CN2 GIA-E 套餐默认的核心节点之一。根据多份公开测评的实测数据：

- **电信方向**：平均延迟在 **161ms** 左右，部分沿海节点能压到 150ms 以内，已经是中美跨太平洋链路的物理下限附近。
- **三网综合平均**：DC6 CN2 GIA-E 三网平均延迟都在 **200ms 以内**，电信最优，联通和移动略高但也在可接受范围。
- **DC9 CN2 GIA**：作为 DC6 的姊妹机房，配置升级为 AMD EPYC + NVMe SSD，路由追踪显示回程同样走 CN2 GIA，延迟表现与 DC6 基本同档。

作为对比，CN2 GT 线路的平均延迟在 **200ms 左右且略有波动**，普通 KVM 直连在晚高峰经常冲到 250ms 甚至 300ms+。这就是 GIA-E 多出来的那部分钱买的东西——**延迟更稳、波动更小**。

### 2. 带宽天花板：2.5Gbps 起跳，高端套餐 10Gbps

搬瓦工 CN2 GIA-E 套餐的链路速度是分档的：

| 套餐档位 | 链路速度 |
|---|---|
| 20G / 40G / 80G | 2.5 Gigabit |
| 160G / 320G | 5 Gigabit |
| 640G 及以上 | 10 Gigabit |

实测中，DC6 CN2 GIA-E 在跑满场景下下载速度可达 **6324.77 Mbps**，上传速度 **5655.33 Mbps**——也就是说 2.5Gbps 的标称带宽是真能跑出来的，不是纸面参数。高端套餐的 10Gbps 链路在带宽充裕时段实测可到 9Gbps 左右。

需要提醒的是：**带宽是共享的**，晚高峰（北京时间 20:00-23:00）会有一定回落，这是所有共享带宽 VPS 的共性，不是搬瓦工独有的问题。

### 3. 丢包率：高端线路在 2026 年表现相当稳定

> "偶发、单机的丢包和大规模的机房异常，是两码事。换个节点测通常就没事了。2026 年的搬瓦工在高端线路上的丢包表现相当稳定。"

这是真实用户长期使用后的反馈。CN2 GIA-E 线路在非极端时段（比如春节、双十一这类全网拥堵节点）丢包率普遍能控制在个位数百分比以内。如果你发现某台机器丢包率突然冲到 30%-50%，第一反应应该是**用 ping.pe 工具换节点复测**，而不是直接判定机房挂了。

普通 KVM 直连和 CN2 GT 在晚高峰的丢包表现会明显差一档，这是线路定位决定的，不是机器问题。

## 三、香港和大阪：低延迟玩家的专属选项

如果你对延迟的要求是"必须低于 100ms"，那 CN2 GIA-E 的 160ms 显然不够看，这时候要考虑物理位置更近的机房。

### 香港 CN2 GIA（HKHK_8 机房）

香港机房走的是三网强制回程 CN2 GIA，延迟数据非常漂亮：

- **电信**：海南 25.8ms，新疆 95.08ms，全国平均约 **49.2ms**
- **移动**：上海 35.86ms，新疆 100.34ms，全国平均约 **62.1ms**
- **联通**：略高于电信和移动，但整体也在 60-80ms 区间

带宽只有 1Gbps，流量也少（最低 500GB/月），但延迟是真的低。适合做对响应速度极敏感的场景，比如低延迟代理、跨境 API 中转、实时音视频中继。

### 大阪 CN2 GIA（JPOS_1 机房）

大阪机房走优化过的 CN2 GIA 线路，带宽约 1.5Gbps，定位和香港类似——用物理距离换延迟。日本方向对国内联通用户友好，晚高峰表现比香港略稳一些（香港偶尔会有国际出口波动），但电信用户延迟会比香港高一点。

## 四、自己动手测：搬瓦工网络性能测试方法

光看别人测的数据不算数，你拿到机器之后一定要自己复测一遍。以下是搬瓦工官方和社区公认的几个测试工具：

1. **ping.pe**：官方推荐的丢包率测试工具，输入 IP 一键测全国三网丢包情况，最快判断线路健康度。
2. **bandwagonhost 官方测速页**：搬瓦工官网提供了所有数据中心的测试 IP 和下载文件地址，下单前可以先 ping 测试 IP 看延迟。
3. **backtrace 三网回程路由测试脚本**：社区脚本，能直观看到回程走的是 CN2 GIA、CMIN2 还是普通直连，是判断"是不是真 GIA"的关键工具。
4. **speedtest-cli / bench.sh**：服务器端测带宽和综合性能的标准脚本，跑一次能拿到 CPU、内存、磁盘、带宽、流媒体解锁等一整套数据。

一个建议：**下单前先测 IP，下单后再测机器**。搬瓦工支持后台自助迁移机房（CN2 GIA-E 套餐可在 15+ 机房间免费迁移），所以就算第一次选的机房延迟不理想，也不用重新买，直接后台切机房就行。

**Q2：为什么我的 CN2 GIA-E 白天 160ms，晚上冲到 250ms？**
晚高峰中美方向国际出口拥堵是常态，所有跨太平洋线路都会受影响。可以尝试迁移到 DC9 CN2 GIA 或 EUNL_9（荷兰 9929）看看是否改善。

**Q4：限量版套餐和常规 CN2 GIA-E 有什么区别？**
限量版（如 $49.99/年的 CN2 GIA-E 10G 限量版）配置更低、流量更少、不定期补货，但走的是同一条 CN2 GIA-E 线路，网络性能没区别。适合预算极紧、能抢到货的用户。

**Q5：普通 KVM 套餐的 DC3 CN2 机房算 CN2 GIA 吗？**
